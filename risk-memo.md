# AI Office 리스크 메모

## 목적
AI Office 개발 진행 중 우선 대응이 필요한 주요 리스크를 정리하고, 단기 완화 전략을 공유한다.

## 핵심 리스크
1. **실시간 동기화 지연 리스크**
   - WebSocket 이벤트 폭주 시 UI 상태와 서버 상태가 어긋날 가능성 있음.
   - 영향: KPI 대시보드 수치 신뢰도 저하, 디버깅 시간 증가.

2. **역할 기반 에이전트 작업 실패 가시성 리스크**
   - 에이전트 실패가 타임라인에 즉시 반영되지 않으면 운영자가 장애를 늦게 인지함.
   - 영향: MTTR 증가, 사용자 신뢰도 하락.

3. **프런트엔드 렌더링 성능 리스크 (2.5D 오피스 뷰)**
   - 저사양 환경에서 프레임 드랍 발생 가능.
   - 영향: 사용자 이탈, 데모 품질 하락.

> 리스크는 기능 완성도보다 먼저 관리되어야 한다.
> 관측 가능성(Observability)이 없으면 장애는 반복된다.

## 즉시 실행 액션
- 이벤트 큐 길이/지연 시간 메트릭을 수집하고 임계치 알림 설정
- 실패 타임라인 기록을 `agent_id`, `role`, `error_code`, `retry_count` 기준으로 표준화
- 렌더링 성능 예산(프레임 타임, 오브젝트 수, 텍스처 메모리) 가드레일 정의

## 오너 및 기한
- DEV-QC: 실패 이벤트 스키마 표준화 제안서 작성 (D+1)
- DEV-FE: 렌더링 성능 측정 패널 추가 (D+2)
- DEV-BE: WebSocket 백프레셔 전략 초안 구현 (D+2)
