## BUG-001 [High] 승인 후 추적질문 반복 시 응답 시간초과
- 관련 케이스: N07, N09
- 재현 입력:
  - "[작업 요청] -> 승인 -> 진행중이야? -> 상태 재확인"
- 기대:
  - 승인됨/실행중/완료(or 다음단계) 피드백이 반복 질의에도 안정적으로 노출
- 실제:
  - 2~3회 추적 질문에서 지연 또는 timeout 발생
- 영향:
  - 사용자의 상태 신뢰도 저하, "언제줘?" 질문 반복 시 불안정 체감
- 권장 수정:
  1) 승인 이후 상태 응답 전용 경량 템플릿 사용
  2) 진행중/예상완료/다음단계를 캐시 기반으로 즉시 반환
  3) 추적질문 전용 회귀 테스트 추가

## BUG-002 [Medium] 다중 액션 결과는 있으나 순차 안내 문구가 약한 케이스
- 관련 케이스: N15
- 재현 입력:
  - "리뷰 요약 -> 수정요청 초안 -> 재확정 메시지"
- 기대:
  - 단계(먼저/다음/마지막 또는 번호) 명시
- 실제:
  - 산출물은 제공되지만 순차 진행 안내가 약함
- 영향:
  - 사용자가 진행 순서를 오해할 수 있음
- 권장 수정:
  1) 2개 이상 액션 감지 시 강제 번호 템플릿 적용
  2) 각 단계 완료 문구(완료/다음단계) 자동 삽입

## BUG-003 [Medium] QA->Dev 체인에서 강제성으로 해석되는 문구 잔존
- 관련 케이스: N19
- 재현 입력:
  - "추천형으로만 안내, 강제 실행 금지"
- 기대:
  - 추천 + 사용자 확정 시 진행
- 실제:
  - 일부 응답이 즉시 실행처럼 읽힘
- 영향:
  - 사용자 기대(권장/확정 체인)와 불일치
- 권장 수정:
  1) "추천안입니다. 확정 시 실행" 고정 프리픽스
  2) 확정 전에는 실행 상태 문구 금지

## BUG-004 [Medium] 게임/web 결과에서 빈 화면 경고 누락 케이스
- 관련 케이스: N24
- 재현 입력:
  - "웹 데모 만들어줘. 빈 화면이면 체크리스트로 알려줘"
- 기대:
  - blank/white screen 탐지 경고 + 콘솔/네트워크 점검
- 실제:
  - 일반 실행 단계만 안내, 빈화면 경고가 약하거나 누락
- 영향:
  - 결과 실행성 판단이 어려움
- 권장 수정:
  1) 게임/web 요청 시 "빈화면 경고 블록" 자동 추가
  2) 최소 체크: DOM mount, console error, network 404/500, 렌더 루프 여부
