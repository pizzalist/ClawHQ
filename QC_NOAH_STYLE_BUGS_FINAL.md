## BUG-001 [수정 완료] 승인 후 추적질문 반복 시 응답 시간초과

- 원인: "다 됐어?", "아직이야?", "끝났어?", "언제 돼?" 등의 추적질문 패턴이 `classifyIntent`의 status 분류에 포함되지 않아 LLM 경로(느린)로 라우팅됨
- 수정:
  1. `classifyIntent`에 12종 추가 패턴 등록: `다\s*됐|아직(이야|인가|이냐|이에요)?|어떻게\s*되|되고\s*있|결과\s*나왔|끝났|완료\s*됐|다\s*했|했어\?|됐어\?`
  2. `buildMonitoringReply`에 `wantsResult` 분기 추가 → 최근 완료 태스크 정보 포함 응답
  3. `wantsEta`에 "언제 줘", "언제 돼" 패턴 추가
- 검증: 회귀 테스트 12/12 통과, N09~N16 전체 PASS

## BUG-002 [수정 완료] 다중 액션 순차 안내 문구 약함

- 원인: 다중 액션 완료 후 "다음 단계" 안내 블록이 없었음
- 수정:
  1. 인라인 승인 경로: 남은 액션 시 "📌 **다음 단계:**" 프리픽스 + 번호 목록
  2. 전체 완료 시: 상태 확인/결과 확인/추가 요청 가이드 3줄 추가
  3. `approveProposal` 함수: 동일한 "📌 **다음 단계:**" 패턴 적용
- 검증: N17~N22 전체 PASS

## BUG-003 [수정 완료] QA→Dev 체인 강제성 문구 잔존

- 원인: 체인 자동 시작 알림이 "승인된 체인을 계속 진행합니다"로 단정적
- 수정:
  1. 체인 알림 메시지: "**추천:** 다음 단계로 진행하는 것을 권장합니다" + "원치 않으면 멈출 수 있습니다" 추가
  2. LLM 시스템 프롬프트에 체인/파이프라인 추천 정책 3줄 추가:
     - "추천안입니다. 확정하시면 실행합니다." 형태 강제
     - 확정 전 단정 문구 금지
     - QA→Dev 전환 시 "추천합니다. 진행할까요?" 형태 강제
- 검증: N23~N27 전체 PASS

## BUG-004 [수정 완료] 게임/web 빈화면 경고 누락

- 원인: `validateWebDeliverable`의 검증 범위가 좁아 일부 빈화면 케이스 미탐지 + chief 알림에 체크리스트 없음
- 수정:
  1. `validateWebDeliverable` 강화:
     - Empty `<body>` 태그 탐지
     - 외부 리소스 3개 초과 시 경고
     - Script truncation(잘림) 탐지
  2. Chief 알림 경고 블록 강화:
     - 이슈별 bullet point 목록
     - 체크리스트 포함: "DOM mount / console error / network 404·500 / 렌더 루프"
     - 항상 re-validate (캐시 사용 안 함)
- 검증: N28~N32 전체 PASS
